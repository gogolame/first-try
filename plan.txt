Порядок в котором выполняется поставленная задача я вижу так:
1) Пользователь заполняет данные на странице в понятном для себя виде(количество управляемых и управляющих устройств, комнат, их параметры).
2) Далее эти данные отправляются на сервер, который должен по определённым правилам проанализировать информацию. Это означает что в web-форме(django - views), которую заполняет 
пользователь нужно заранее продумать и сгруппировать типы устройств, может быть выделить их специфику(потому что например шторы и лампочка отличаются по функционалу), типы и 
размеры комнат. О том как я это вижу показываю в скриншоте, приложенным к письму. 
3)Попав на сервер данные обрабатываются, и из них конфигурируются файлы типа *.items и *.sitemap, необходимые для openHAB. Этот пункт стоит выделить потому что именно здесь 
необходимо создать правильные POST-запросы, которые будут вытягивать данные из формы. Как я понял 16-бит адреса каналов так же должны назначаться с помощью кода, а не в ручную.
Структура файлов типа *.items - "itemtype itemname ["labeltext"] [<iconname>] [(group1, group2, ...)] [["tag1", "tag2", ...]] [{bindingconfig}]"
Пример структы файлов типа *.sitemap 
-------------------------------------------------------

sitemap demo label="My home automation" {
    Frame label="Date" {
        Text item=Date
    }
    Frame label="Demo" {
        Group item=Heating
        Switch item=Lights icon="big_bulb" mappings=[OFF="All Off"]
        Text item=Temperature valuecolor=[>25="orange",>15="green",<=15="blue"]
        Text item=Multimedia_Summary label="Multimedia" icon="video" {
            Selection item=TV_Channel mappings=[0="off", 1="DasErste", 2="BBC One", 3="Cartoon Network"]
            Slider item=Volume
        }
    }
}

------------------------------------------------------


Планы относительно задачи:
1) Работать собираюсь на django
2) Создать тестовый пример на своей машине, в котором будет простая view, необходимая для POST-запроса
3) Разобраться со следующими вещами: формы в html, django-шаблон html формы, django forms, django модели, миграции
4) Научиться в конечном итоге получать файлы типа *.items и *.sitemap из моей тестовой формы.
